## 摘要

本报告分析了现有 prosody.py 的韵律曲线参数（speed/volume/pause）与 MiMo-V2.5-TTS 风格控制能力之间的映射关系。核心结论：MiMo 通过**两层控制机制**（user 角色的自然语言风格指令 + assistant 角色的音频标签）可以完整表达 prosody curve 的渐变效果，但需要将数值型参数转换为语义化的自然语言描述。

## 详细发现

### [RESEARCH] 1. 现有 Prosody Curve 系统分析

**来源**: `prosody.py`, `config.py` L402-408

#### 1.1 Hypnotic 曲线配置

```python
PROSODY_CURVES = {
    "hypnotic": {
        "speed":  [(0.0, 1.0), (0.3, 0.9), (0.7, 0.75), (1.0, 0.55)],
        "volume": [(0.0, 1.0), (0.5, 0.85), (0.8, 0.6),  (1.0, 0.3)],
        "pause":  [(0.0, 0.3), (0.5, 0.6),  (0.8, 1.2),  (1.0, 2.0)],
    },
}
```

三条分段线性曲线，以 progress (0.0→1.0) 为自变量：
- **speed**: 1.0 → 0.55（最终语速降至初始的 55%）
- **volume**: 1.0 → 0.3（最终音量降至初始的 30%）
- **pause**: 0.3s → 2.0s（句间停顿从 0.3s 增至 2.0s）

#### 1.2 阶段标记系统

**来源**: `prosody.py` L38-42

```python
PHASE_ANCHORS = {
    "引入": 0.0,
    "深入": 0.3,
    "尾声": 0.7,
}
```

阶段标记 `[阶段：引入/深入/尾声]` 将脚本进度锚定到固定点，实现非均匀进度分布。`_build_progress_map()` 函数（L74-118）根据阶段锚点重映射每个 speech block 的 progress 值。

#### 1.3 内联标签乘法系数

**来源**: `prosody.py` L45-49

```python
TAG_MULTIPLIERS = {
    "慢速": {"speed": 0.8, "volume": 1.0},
    "轻声": {"speed": 1.0, "volume": 0.5},
    "极弱": {"speed": 0.7, "volume": 0.3},
}
```

内联标签 `[慢速]`/`[轻声]`/`[极弱]` 是**乘法系数**，作用于曲线插值后的 base 值。同一标签在不同进度点产生不同绝对效果（如进度 0.8 时 `[慢速]` 的 speed = 0.75 * 0.8 = 0.6）。

#### 1.4 TTS 调用逻辑

**来源**: `engine.py` L631-676

当前 TTS 流程：
1. `apply_curve_to_blocks(blocks, curve)` — 为每个 speech block 注入 speed/vol/progress
2. 循环中读取 `block["speed"]` 和 `block["vol"]`
3. CosyVoice: `_synthesize_cosyvoice(text, path, speed=speed)`
4. edge-tts: `edge_rate = f"{int((speed - 1.0) * 100):+d}%"` — 将 speed 转换为百分比 rate
5. 后处理: `clip.volumex(vol)` — 应用音量系数

### [RESEARCH] 2. MiMo TTS 风格控制能力

**来源**: `.plans/mimo-tts-integration/docs/api-contracts.md`

#### 2.1 两层控制机制

MiMo API 通过 messages 数组实现两层风格控制：

| 层级 | role | content 格式 | 控制粒度 | 示例 |
|------|------|-------------|---------|------|
| 全局风格 | user | 自然语言描述 | 整段/整篇 | "温柔慵懒的低音，语速缓慢" |
| 局部标签 | assistant | `(标签)文本内容` | 单句/单段 | "(温柔)你好，今天过得怎么样？" |

#### 2.2 风格标签类型

基于 MiMo TTS 的设计模式，音频标签放在 assistant content 中：

| 标签类型 | 语法 | 效果 |
|---------|------|------|
| 情感标签 | `(温柔)` `(悲伤)` `(平静)` | 控制情感基调 |
| 语速标签 | `(缓慢)` `(快速)` | 控制语速 |
| 音量标签 | `(轻声)` `(低语)` `(大声)` | 控制音量 |
| 组合标签 | `(温柔缓慢低语)` | 多维度同时控制 |

#### 2.3 导演模式（Director Mode）

导演模式通过三维度精细控制：

```
角色：温柔的睡前故事讲述者，声音低沉磁性
场景：深夜安静的卧室，只有台灯的微光
指导：语速从正常逐渐变慢，音量从清晰逐渐减弱到耳边低语
```

三维度说明：
- **角色 (Character)**: 声音人格（音色、年龄、性格）
- **场景 (Scene)**: 环境氛围（空间、时间、光线）
- **指导 (Direction)**: 具体演绎指令（语速、音量、情感变化）

### [RESEARCH] 3. 映射方案设计

#### 3.1 Progress → MiMo 风格标签（user role content）

将 progress 范围映射为语义化的风格描述：

| Progress | Phase | Speed | Volume | MiMo Style (user content) |
|----------|-------|-------|--------|---------------------------|
| 0.0-0.15 | 引入初段 | 1.0→0.95 | 1.0→0.98 | "自然平静的语调，正常语速，清晰温暖" |
| 0.15-0.30 | 引入末段 | 0.95→0.90 | 0.98→0.95 | "稍微放慢语速，声音温暖柔和" |
| 0.30-0.50 | 深入前半 | 0.90→0.82 | 0.95→0.80 | "轻柔缓慢的低语，逐渐放松" |
| 0.50-0.70 | 深入后半 | 0.82→0.75 | 0.80→0.60 | "缓慢低沉的耳边呢喃，声音越来越轻" |
| 0.70-0.85 | 尾声前半 | 0.75→0.65 | 0.60→0.45 | "极轻极慢的催眠低语，慵懒欲睡" |
| 0.85-1.00 | 尾声后半 | 0.65→0.55 | 0.45→0.30 | "几乎听不见的气声，像在梦中呢喃" |

#### 3.2 内联标签 → MiMo 音频标签（assistant role content）

| 现有内联标签 | MiMo 音频标签 | 说明 |
|-------------|--------------|------|
| `[慢速]` | `(缓慢)` | 语速降低 |
| `[轻声]` | `(轻声)` | 音量减弱 |
| `[极弱]` | `(极轻低语)` | 语速+音量双重减弱 |
| `[停顿]` | 保持原样处理 | MiMo 通过标点或 SSML 控制停顿 |
| `[环境音：xxx]` | 不传给 MiMo | 由后处理插入静音轨 |

#### 3.3 阶段标记 → 导演模式指令

阶段标记对应导演模式的「指导」维度：

| 阶段标记 | Progress 锚点 | 导演模式指导 |
|---------|--------------|-------------|
| `[阶段：引入]` | 0.0 | "开始讲述，语速正常，声音温暖清晰，像在轻声讲故事" |
| `[阶段：深入]` | 0.3 | "逐渐放慢语速，降低音量，营造放松氛围，像在耳边低语" |
| `[阶段：尾声]` | 0.7 | "语速极慢，音量极轻，用气声呢喃，引导听众进入睡眠" |

#### 3.4 实现方案：分段合成策略

**核心问题**: MiMo API 没有逐句控制语速/音量的数值参数，只有语义化的风格描述。

**推荐方案**: 按 progress 区间分段，每段使用不同的 user content 风格指令。

```
脚本分段:
  [段1] progress 0.0-0.3  → user: "自然平静的语调，正常语速"
  [段2] progress 0.3-0.7  → user: "轻柔缓慢的低语，逐渐放松"
  [段3] progress 0.7-1.0  → user: "极轻极慢的催眠低语"

每段内的 speech block:
  - 基础风格由段的 user content 控制
  - 内联标签通过 assistant content 的音频标签微调
  - 停顿由标点或 API 参数控制
```

#### 3.5 精细度权衡

| 方案 | 精细度 | API 调用次数 | 延迟 | 推荐场景 |
|------|--------|-------------|------|---------|
| 三段式（引入/深入/尾声） | 低 | 3 次 | 低 | 快速原型 |
| 六段式（每段 ~15%） | 中 | 6 次 | 中 | **推荐** — 平衡质量与性能 |
| 逐句式 | 高 | N 次 | 高 | 不推荐 — 延迟过高，风格不连贯 |

## 结论与建议

### 核心结论

1. **Prosody curve 可完整映射到 MiMo**: speed/volume/pause 三维度都能通过 MiMo 的自然语言风格指令表达，但需要数值→语义的转换层。

2. **两层控制机制天然匹配**: user content 承载 progress-based 全局风格，assistant content 承载 inline tag 局部微调。

3. **精度损失可接受**: prosody curve 是连续数值插值（如 speed=0.82），MiMo 是语义区间（"轻柔缓慢"），但对于催眠/助眠场景，听感差异极小。

4. **分段策略是关键**: 推荐六段式分段，每段一个 user content 风格指令，段内通过音频标签微调。

### 建议方案

1. **在 prosody.py 新增 `ProsodyCurve.to_mimo_styles()` 方法**:
   - 输入: speech block 列表（含 progress）
   - 输出: 每个 block 对应的 `{"user_style": str, "audio_tag": str}`

2. **分段策略**: 按 progress 将脚本分为 6 段，每段约 15-20%，段边界对齐阶段标记。

3. **风格描述模板**: 在 config.py 新增 `MIMO_STYLE_TEMPLATES` 配置，按 progress 区间定义语义化描述。

4. **内联标签映射表**: 在 config.py 新增 `MIMO_AUDIO_TAGS` 配置，映射现有内联标签到 MiMo 音频标签。

5. **导演模式可选增强**: 对于 Voice Design 音色，可在 user content 中附加导演模式的三维度描述（角色+场景+指导），增强表现力。

### 风险/缺口

1. **MiMo 风格标签的实际效果未验证**: 以上映射基于 API 设计推测，需要实际合成测试确认听感。

2. **停顿控制不确定性**: MiMo 是否支持精确的停顿时长控制（如 `[停顿500ms]`）需确认，可能需要在后处理中插入静音。

3. **风格切换的连贯性**: 频繁切换 user content 可能导致声音特征突变，需要测试六段式方案的过渡是否自然。

4. **API 延迟**: 六段式意味着 6 次 API 调用，需评估总延迟是否可接受（并行调用可能缓解）。
